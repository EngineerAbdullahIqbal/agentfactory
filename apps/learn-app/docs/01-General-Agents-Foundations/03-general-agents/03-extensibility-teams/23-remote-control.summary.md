Remote Control lets you continue a local Claude Code session from any device -- your phone, tablet, or any browser -- by connecting through claude.ai/code or the Claude mobile app. Start a session with `claude remote-control` (fresh start) or `/rc` (existing session), then connect by opening the session URL, scanning the QR code (press spacebar to display it), or finding the session by name in claude.ai/code. The critical architectural point is that code runs entirely on your local machine: your filesystem, MCP servers, tools, and project configuration all stay available. The Anthropic API acts purely as a message relay using outbound-only HTTPS connections with TLS and short-lived credentials -- no inbound ports, no firewall changes, no cloud execution. This makes it architecturally similar to a CI/CD self-hosted runner where your infrastructure does the work and the server just routes instructions. Remote Control requires a Pro or Max plan (not available on Team, Enterprise, or API keys) and supports one remote session per Claude Code instance.

The most important practical technique is the tmux survival trick: start `claude remote-control` inside a tmux session, detach with Ctrl+B then D, and the Claude process survives even after closing your terminal entirely. Without tmux, closing the terminal kills the local process and ends the session. The roughly 10-minute network timeout is another limitation to understand -- if your machine is awake but offline for more than 10 minutes, the session exits. For choosing between access methods, the decision framework is straightforward: use Remote Control when you need your local environment accessible from another device, use Claude Code on the Web (Lesson 25) when you want to work without any local machine at all, and use your terminal when you are at your desk with full access. Combining Remote Control with worktrees (Lesson 22) creates the power combo of isolated branch work plus persistent processes plus mobile monitoring -- each piece solves a different problem, and together they enable resilient parallel feature development from anywhere.
