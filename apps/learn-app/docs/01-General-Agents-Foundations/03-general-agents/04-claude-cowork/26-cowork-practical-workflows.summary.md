### Core Concept

Cowork's value is not just speed—it makes tasks feasible that you would otherwise skip entirely, turning multi-hour manual processes into minutes-long conversations by following four universal workflow patterns.

### Key Mental Models

- **Feasibility Shift, Not Just Speed**: The real gain is tasks moving from "never gets done" to "done in minutes." Downloads cleanup at 24x faster matters less than Downloads cleanup finally happening.
- **Outcome Specification Over Step Instruction**: Effective Cowork prompts describe the desired result (format, structure, quality criteria) rather than enumerating every step—Claude handles the implementation.
- **Workflow Design as a Repeatable Skill**: Any repetitive task that involves files or documents can be designed into a reusable Cowork workflow using four steps: identify, clarify outcome, add constraints, review and refine.

### Critical Patterns

- Always specify output format explicitly in your prompt—structure like "executive summary, top 10 table, notable changes" produces consistent, usable results.
- Use the four workflow patterns as a mental checklist: Explore First, Propose Then Execute, Handle Variation, Report Results.
- Start with a single, simple workflow before chaining multiple steps—build complexity after the first version succeeds.
- Cowork supports scheduled recurring tasks, turning one-off workflows into persistent automation.

### Common Mistakes

- Giving vague instructions ("clean up this folder") instead of specifying the target outcome ("organize by type into: documents, images, archives, installers")—vagueness produces vague results.
- Skipping the approval review for deletion or modification operations—the propose-then-execute cycle exists precisely to catch mistakes before they happen.
- Jumping to complex multi-step workflows before mastering single-step operations—each layer of complexity multiplies the chance of unexpected results.
