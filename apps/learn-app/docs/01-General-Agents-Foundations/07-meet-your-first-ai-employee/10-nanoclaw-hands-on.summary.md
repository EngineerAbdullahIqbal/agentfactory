This lesson is the hands-on companion to Lesson 9's conceptual architecture. Students install NanoClaw using Docker and Claude Code (git clone, cd, claude, /setup -- under 5 minutes), connect WhatsApp, and run the same research and file creation tasks from Lesson 3 on a container-isolated platform. The side-by-side experience with OpenClaw makes architectural differences tangible: files live inside containers instead of the shared filesystem, the security default inverts from "everything accessible" to "nothing accessible," and Claude Code can load NanoClaw's entire ~500-line codebase into context.

The lesson covers four key skills: **container-based agent setup** (Docker pre-flight and NanoClaw installation), **container isolation verification** (docker ps and understanding what stays inside the boundary), **platform comparison analysis** (structured comparison across setup time, security model, and codebase size), and **Intelligence Layer design** (the chapter capstone exercise). Students experience the 35K token advantage firsthand by asking NanoClaw to describe its own codebase, and they verify container isolation by examining running containers with docker ps.

The capstone exercise requires students to design Layer 3 (Intelligence) for their own profession -- listing Agent Skills (domain knowledge), MCP servers (domain tools), and container isolation requirements. This is where every Chapter 7 concept converges: universal patterns, skills portability, security architecture, and the Agent Factory vision. Three "Try With AI" prompts guide students through platform evaluation, design iteration, and container boundary analysis for their specific domain.
