This lesson packages the entire IDFA methodology — three layers, four guardrails, naming conventions, the Agent Decision Table, trigger phrases, and common mistakes — as a portable Agent Skill following the agentskills.io open standard. The complete SKILL.md file is presented for students to copy and install, with installation paths for **Claude Code** (.claude/skills/idfa-financial-architect/SKILL.md), **GitHub Copilot** (.github/copilot-instructions.md), **VS Code** (workspace custom instructions), and **Codex** (project instructions). Once installed, the skill transforms agent behaviour: every financial modelling interaction automatically applies Named Range Priority, LaTeX Verification, Intent Notes, and MCP Dependency without explicit prompting.

The lesson teaches students to verify skill activation by testing with a new intent statement and checking whether all four guardrails appear in the output. The **Agent Decision Table** maps seven categories of financial modelling tasks to their correct IDFA workflows, while **trigger phrases** show how natural language like "I inherited this model" or "this model is a black box" activates the appropriate IDFA response. Five common mistakes are embedded in the skill as safeguards that the agent checks its own output against.

The business bottom line is portability: one file, every agent, every model interaction governed by the same four guardrails. The IDFA SKILL.md demonstrates what it looks like when a domain methodology is packaged as a skill rather than memorised as a process — connecting back to the plugin architecture from Chapter 15 and extending the value of everything learned in Lessons 1 through 8 across every skills-compatible tool.
