**Core Concept.** Every IDFA-compliant financial model has exactly three layers that must remain functionally isolated: Layer 1 (Assumptions) holds inputs only as Named Ranges with the `Inp_` prefix, Layer 2 (Calculations) holds formulas that reference only Named Ranges with zero cell addresses, and Layer 3 (Output) holds presentation and formatting that reads from Layer 2 only. The isolation rule — changing a Layer 1 input can never break a Layer 2 formula — is the architectural guarantee that eliminates Formula Rot.

**Key Mental Models.** Named Ranges are the mechanism that enforces layer separation: a formula reading `Inp_Rev_Y1` survives any structural change because it references a name, not a position. The naming convention (Inp\_ for inputs, Variable_Yn for annual calculations, Variable_Total for aggregates, Variable_Pct for ratios) is not cosmetic — it is the system that lets any reader instantly distinguish inputs from derived values. Intent statement decomposition — extracting every assumption from prose before writing any formula — is the first step of every IDFA model build and the foundation of the Intent Synthesis capability covered in Lesson 11.

**Critical Patterns and Common Mistakes.** The most common violation is embedding a constant in a Layer 2 formula (e.g., `=Revenue_Y1 * 0.60` instead of `=Revenue_Y1 * Inp_COGS_Pct_Y1`). The second is placing a formula in Layer 1, which should contain only raw input values. The third is reading from Layer 1 directly in Layer 3, bypassing the Calculations layer. Each violation breaks the isolation rule and reintroduces the fragility that IDFA exists to eliminate. The three layers can live on separate tabs or separate sections of one tab — physical layout is flexible, but functional isolation is not.
