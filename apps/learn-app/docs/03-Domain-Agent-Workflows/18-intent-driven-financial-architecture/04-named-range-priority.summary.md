**Core Concept.** Guardrail 1 — Named Range Priority — requires that every formula in the Calculation layer references only Named Ranges, never cell coordinates. The compliance test is immediate: if you can understand what a formula calculates without clicking on any referenced cell, it passes. This single rule eliminates silent breakage from row insertion, makes every formula self-documenting as a business rule, and enables AI agents to interpret model logic without reverse-engineering cell addresses.

**Key Mental Models and Critical Patterns.** The twelve GP Waterfall formulas follow exactly two patterns: direct assignment from an Assumptions input (Revenue*Y1 = Inp_Rev_Y1) and arithmetic on previously defined Named Ranges (Gross_Profit_Y2 = Revenue_Y2 - COGS_Y2). Assumptions inputs use the Inp* prefix; Calculation results use Variable_Dimension format. Any hardcoded constant in a formula — even if the formula also uses Named Ranges — violates layer isolation and must be moved to the Assumptions layer. Named Ranges cannot contain spaces; underscores are the universal separator.

**Common Mistakes and Connections.** The most frequent violation is embedding a constant like 0.60 directly in a Calculation formula instead of referencing Inp_COGS_Pct_Y1 from the Assumptions layer. This creates a second source of truth that will silently diverge when someone updates the assumption. Guardrail 1 is the foundation for all other IDFA guardrails — LaTeX Verification, Intent Notes, and MCP Dependency all require Named Range compliant formulas to function. The Calculation layer built in this lesson is extended in L05 with complex formulas (WACC, NPV) and in L06 with audit-ready Intent Notes.
