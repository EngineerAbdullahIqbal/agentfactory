This capstone lesson validates all five Finance Domain Agent capabilities on the GP Waterfall model built throughout the chapter, extended with Operating Expenses and EBITDA. The five capabilities — **Intent Synthesis**, **Deterministic What-If**, **Logic De-compilation**, **Strategic Goal-Seeking**, and **Stochastic Simulation** — each test a specific aspect of what IDFA-compliant architecture enables. Every test follows the same principle: the agent reasons, MCP mediates, and Excel calculates.

The lesson walks through each capability test with a defined procedure and pass criteria. Intent Synthesis verifies the agent can convert plain English into a complete IDFA specification before touching Excel. Deterministic What-If confirms every reported number comes from `read_cell()`, not internal arithmetic. Logic De-compilation proves the agent can reconstruct business rules from formulas alone. Strategic Goal-Seeking validates iterative input search via MCP. Stochastic Simulation orchestrates Monte Carlo scenarios where every iteration runs through the actual model.

Pass all five and the IDFA deployment is production-ready. Fail any one and the failure traces to a specific architectural issue — missing Named Ranges, internal agent arithmetic, incomplete model inspection, unreliable MCP iteration, or insufficient iteration control. These tests are the formal proof that the methodology delivers what it promises.
