### Core Concept
Practice exercises close the gap between understanding database workflows and executing them under pressure. Each module pairs a Build exercise (create from scratch) with a Debug exercise (diagnose broken code).

### Continuity Bridge
From capstone integration to independent skill demonstration across all six database development workflows: modeling, CRUD, relationships, transactions, cloud deployment, and hybrid verification.

### Key Mental Models
- Build + Debug pairing develops two distinct skills: applying workflows correctly vs diagnosing someone else's mistakes.
- The Database Development Framework (Model, Connect, Operate, Protect, Verify, Deploy) applies to every exercise.
- Scaffolding is deliberately removed: Modules 1-3 have starter prompts, Modules 4-6 remove them, Capstones remove everything.

### Critical Patterns
- Module 1: Data Modeling — Library Catalog (Build) + Broken Pet Store (Debug).
- Module 2: CRUD Operations — Recipe Book (Build) + Broken Task Manager (Debug).
- Module 3: Relationships — Music Library (Build) + Broken Blog (Debug).
- Module 4: Transactions — Game Inventory Trading (Build) + Broken Bank (Debug).
- Module 5: Cloud Deployment — Contact Book Deploy (Build) + Connection Doctor (Debug).
- Module 6: Hybrid Verification — Expense Audit (Build) + Wrong Tool (Analysis).
- Module 7: Capstones — Student Grade Portal, CSV Migration, Disaster Recovery.

### Common Mistake
Fixing multiple bugs at once instead of one at a time. In Debug exercises, fixing one bug and re-running tests isolates each problem; batch-fixing makes it impossible to know which fix resolved which failure.
